## Four Pillars of Features

UpWord's article reader is organized around **four core feature pillars**:

<CardGrid stagger>
	<Card title="Reading" icon="open-book">
		Difficulty switching (Easy/Medium/Hard) and reading progress tracking.
	</Card>
	<Card title="Syntax" icon="pencil">
		Grammar analysis with visual annotations and syntax highlighting.
	</Card>
	<Card title="Echoes" icon="approve-check-circle">
		Cross-article context display when hovering over vocabulary words.
	</Card>
	<Card title="Audio" icon="rss">
		TTS preloading, audio player, and Edge TTS client support.
	</Card>
</CardGrid>

## Tech Stack

<CardGrid>
	<Card title="Frontend" icon="laptop">
		**Astro** + React components, deployed on Cloudflare Pages.
	</Card>
	<Card title="Backend" icon="seti:db">
		**Elysia** (Bun) + SQLite/D1/Turso, deployed via Docker.
	</Card>
	<Card title="LLM Pipeline" icon="star">
		Multi-stage article generation with difficulty adaptation.
	</Card>
	<Card title="Task Queue" icon="list-format">
		Background workers for async LLM processing.
	</Card>
</CardGrid>
